# 02 分层模型

## 分层职责

### core
用于稳定的、跨项目、跨平台行为规则。

### platform
仅用于依赖特定运行时/工具链的行为。

### learnings
用于复盘产物：可复用经验、重复错误模式、改进请求。

### rollout_summaries
用于完整回合复盘、带证据的上下文，以及仍需更高置信度判断的待晋升结论。

### memory-sidecar
用于仅运行时需要的材料：原始证据、session 级压缩状态、轻量 current/recent 索引。

## 边界规则

- 不要把临时任务状态写进 `core`。
- 不要把平台特性写进 `core`。
- 不要让 sidecar 里的运行时材料绕过复盘层直接进入 `core`。
- handoff 工件放在项目工作区，不放进记忆层。
- 先按 scope 路由，再按质量评分判断。

## 真源规则

- 先更新 `core`。
- 再同步到 `platform`。
- 同一行为保持一个规范化主条目。
