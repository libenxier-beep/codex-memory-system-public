# 07 最小增量迁移模式

## 目标

从扁平旧记忆文件迁移到分层架构，同时尽量减少扰动。

## 迁移步骤

1. 先做功能盘点
2. 定义分层目标映射
3. 按“行为语义”而非文件名检测重复
4. 按触发条件与 scope 检测冲突
5. 选择更强版本作为单一真源
6. 先做最小改动
7. 记录结构性变更
8. 如果旧材料仍在支撑运行时或复盘语境，先冻结 legacy 源，再决定是否删除

## 冲突策略

每个冲突都给三选一：

- A：保留本地版本
- B：替换为迁入版本
- C：分层合并（通常优先，前提是小改且安全）

## 禁止事项

- 不要为同一行为创建并行系统
- 不要迁移低价值复杂度
- 不要盲目覆盖已稳定运行规则
- 迁移后不要继续把 raw evidence、session compression、retrospective 混放在语义模糊的旧桶里
