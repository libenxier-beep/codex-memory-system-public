# 05 安全与治理

## 写入门禁清单

- Value：是否长期且可复用？
- Scope：是否分层正确？
- Pollution：是否剔除本地一次性细节？
- Safety：是否排除密钥/注入/危险载荷？
- Evidence：是否有可复核依据？

## 存储安全检查

以下内容应拒绝入库或先脱敏：

- 凭证、令牌、私钥、Cookie
- prompt 注入残留、隐藏控制文本
- 不必要的可执行载荷
- 高风险领域中的不当泛化结论

## 治理原则

- 质量优先于数量
- 明确边界优先于宽泛假设
- 可审计优先于方便写入
- 低置信度场景采用保守持久化
- 私人类材料的写入范围应比工作记忆更窄，隐私过滤应更严格
