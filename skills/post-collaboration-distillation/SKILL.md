# Post-Collaboration Distillation

## Overview

Run a conservative distillation pass after meaningful work concludes.
This skill decides what is worth keeping, how strongly it is supported, and which memory layer should receive it.

## Workflow

1. Read the completed round plus any needed sidecar evidence.
2. Summarize what changed and what actually mattered.
3. Extract candidate insights only from validated conclusions, repeated patterns, or strong preference signals.
4. Reject weak, local-only, or noisy material.
5. Compare candidates against existing memory before adding anything new.
6. Route surviving candidates to sidecar, rollout retrospectives, work memory, personal memory, or discard.
7. Produce a Distillation Report.

## Boundary

- Do not replace the primary task.
- Do not invent durable rules from weak evidence.
- Do not bypass deduplication or scope checks.
- Do not commit accepted candidates directly if a downstream commit stage is expected.
