# Routing And Clarification

- `rollout_summaries/` is the default holding layer for accepted items that should not yet be promoted.
- Work-memory commits should route through the smallest relevant truth-source set under `memories/`.
- Personal-memory commits should route through `personal_memory/README.md` and the smallest relevant internal file set.
- Clarification is reserved for meaningful ambiguity where writing to the wrong layer would create privacy or pollution cost.
- Reject-only means the user only needs to say what should not be committed in the current run.
