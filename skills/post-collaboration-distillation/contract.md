# Distillation Contract

## Inputs

- completed task or collaboration context
- relevant runtime evidence when needed
- current memory files for deduplication and routing checks

## Required candidate fields

- `candidate_id`
- `summary`
- `why`
- `scope_hint`
- `recommended_destination`
- `draft_write`

## Outputs

- Distillation Report
- candidate list for downstream review or commit

## Non-goals

- raw transcript archiving
- direct durable promotion without routing and scope checks
