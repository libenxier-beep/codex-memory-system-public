# 02 分层模型

## 分层职责

### core
用于稳定的、跨项目、跨平台行为规则。

### platform
仅用于依赖特定运行时/工具链的行为。

### learnings
用于复盘产物：可复用经验、重复错误模式、改进请求。

### short_term
用于当前任务上下文、候选记忆、仅摘要的会话输出。

## 边界规则

- 不要把临时任务状态写进 `core`。
- 不要把平台特性写进 `core`。
- 不要让 `short_term` 绕过质量门禁直接进入 `core`。
- 先按 scope 路由，再按质量评分判断。

## 真源规则

- 先更新 `core`。
- 再同步到 `platform`。
- 同一行为保持一个规范化主条目。
